# Sales Script

This script provides a structure for consultative sales conversations.

It should be adapted to the prospect’s actual context.

---

## Opening

> Based on our discovery conversation, I want to summarize what I heard, confirm that I understand the problem correctly, and then walk through what an implementation could look like.

---

## Problem confirmation

> The main issue appears to be [problem].  
> This affects [people/process/business area].  
> The cost is showing up as [time, errors, missed follow-up, lost revenue, staff frustration, risk].

Ask:

> Is that accurate?

---

## Current-state summary

> Right now, the workflow looks like this: [current workflow].  
> The biggest friction points are [friction points].

---

## Opportunity framing

> The opportunity is not just to add AI. The opportunity is to make this workflow more reliable, easier to manage, and less dependent on manual follow-up.

---

## Proposed outcome

> The goal would be to create [outcome], so that [business result].

Examples:

- fewer missed follow-ups
- faster admin processing
- clearer task handoff
- better customer response
- less duplicated data entry
- more consistent internal execution

---

## Implementation path

> I would approach this in phases:

1. Map the current workflow.
2. Identify the highest-value automation points.
3. Design the improved workflow.
4. Implement the first working version.
5. Test it with real use.
6. Document the system.
7. Train the team.
8. Review results and next improvements.

---

## Risk framing

> The main risks I see are [risks].  
> I would reduce those by [controls].

---

## Pricing transition

> Based on the scope we discussed, this would fit best as [offer/package]. I will explain what is included, what is not included, and what the next step would be.

---

## Decision question

> Does this feel like the right problem to solve now?

If yes:

> The next step would be [proposal, deposit, implementation kickoff, technical audit].

If no:

> What would need to change for this to become worth solving?
