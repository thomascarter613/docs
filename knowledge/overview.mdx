# Knowledge Graph Overview

The Knowledge Graph turns concepts, patterns, decisions, lineage, and operating memory into structured knowledge.

It helps Applied Innovation Corp and Monad-Workspace connect what is being learned across founder work, platform architecture, delivery, narrative, and governance.

---

## Purpose

The Knowledge Graph exists to:

- define core concepts
- identify recurring patterns
- connect decisions to consequences
- preserve lineage
- support AI-assisted reasoning
- reduce repeated explanation
- make the operating system more coherent

---

## Knowledge sources

The Knowledge Graph draws from:

- Founder Logs
- ADRs
- governance docs
- delivery lessons
- pipeline reviews
- platform requirements
- narrative doctrine
- monthly themes

---

## Core knowledge types

| Type | Purpose |
| --- | --- |
| Concepts | Important terms and definitions |
| Patterns | Repeated structures, behaviors, or lessons |
| Lineage | Relationships between origins, decisions, systems, and outcomes |
| Decisions | Accepted choices that affect future work |
| Assets | Reusable outputs produced by the operating system |

---

## Definition of done

The Knowledge Graph is working when important ideas can be reused without being re-explained from scratch.
