# Patterns Overview

Patterns are recurring structures, behaviors, problems, or lessons that appear across the business.

A pattern is more durable than a single observation.

---

## Purpose

The Patterns system exists to:

- identify repetition
- convert repeated friction into systems
- turn repeated lessons into doctrine
- support platform design
- improve sales and delivery
- help the AI Cofounder reason from evidence

---

## Pattern sources

Patterns may come from:

- Daily Logs
- Weekly Reviews
- Monthly Themes
- sales objections
- client delivery
- platform development
- documentation work
- governance review

---

## Pattern types

| Pattern type | Example |
| --- | --- |
| Sales pattern | Buyers respond better to operational outcomes than AI novelty |
| Delivery pattern | Workflow mapping is needed before automation |
| Platform pattern | Repeated need for context lineage |
| Governance pattern | AI output requires human verification before promotion |
| Narrative pattern | Founder story becomes clearer through build-in-public notes |

---

## Definition of done

The Patterns system is working when repeated lessons become reusable operating intelligence.
