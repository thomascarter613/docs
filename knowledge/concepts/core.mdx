# Core Concepts

Core Concepts are the foundational terms used across Applied Innovation Corp and Monad-Workspace.

They should remain clear, stable, and reusable.

---

## Concept table

| Concept | Definition |
| --- | --- |
| AI Cofounder | The AI-assisted operating partner that helps with planning, drafting, review, extraction, and execution under human verification. |
| Founder Log | The operating memory that captures founder work, decisions, assumptions, risks, and context. |
| Monad-Workspace | The local-first, AI-ready, governance-grade workspace and platform foundation being built by Applied Innovation Corp. |
| ADR | Architecture Decision Record; a durable record of an important architectural or governance decision. |
| Operating Memory | Structured context that helps the business resume, reason, and improve over time. |
| Governance | The system of rules, reviews, standards, and verification that keeps work coherent and safe. |
| Pipeline Engine | The system for converting market insight into qualified business opportunities. |
| Delivery Engine | The system for converting sold work into client outcomes and reusable assets. |
| Narrative Engine | The system for converting internal learning into public clarity and authority. |
| Platform Engine | The system for converting validated needs into product and architecture decisions. |

---

## Concept standard

Each core concept should be:

- clearly named
- defined in plain language
- connected to related systems
- reviewed when the business changes
- usable by humans and AI systems

Core Concepts are working when they reduce ambiguity across the docs.
