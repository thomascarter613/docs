# Positioning Framework

Use this framework to refine positioning for Applied Innovation Corp, Monad-Workspace, and related offers.

---

## Positioning statement

```md
For [target customer],
who struggle with [urgent problem],
Applied Innovation Corp provides [offer/category],
that helps them achieve [business outcome],
unlike [alternative],
because [differentiator/proof].
```

## Positioning fields

| Field           | Question                                             |
| --------------- | ---------------------------------------------------- |
| Target customer | Who is this specifically for?                        |
| Problem         | What painful problem do they already recognize?      |
| Outcome         | What measurable or meaningful result do they want?   |
| Category        | What kind of solution do they think they are buying? |
| Alternative     | What would they do instead?                          |
| Differentiator  | Why is this approach meaningfully different?         |
| Proof           | What evidence supports the claim?                    |

## Message hierarchy

Positioning should move from simple to detailed:

1. One-line explanation
2. Problem statement
3. Outcome statement
4. Offer explanation
5. Differentiator
6. Proof
7. Deeper doctrine

## Evaluation questions

Ask:

* Would the buyer understand this immediately?
* Does this use buyer language?
* Does this describe a painful problem?
* Does this avoid vague AI hype?
* Does this explain the business outcome?
* Is the differentiator believable?
* Is the proof real?

## Rule of thumb

If the positioning requires too much explanation, it is not yet clear enough.
