# Promotion Workflow

The Promotion Workflow explains how raw Founder Log entries become reusable business assets.

A Founder Log entry should not stay trapped as a note if it contains durable value. It should be promoted into the system where it can improve future execution.

---

## Purpose

Promotion turns founder memory into operating infrastructure.

A log entry may become:

- a decision
- an assumption
- a risk
- a task
- a template
- an SOP
- a sales script
- a pricing rule
- a delivery workflow
- a product requirement
- a narrative asset
- a governance rule
- a knowledge graph concept

The goal is to prevent useful insight from being lost.

---

## Promotion triggers

Promote a Founder Log item when it is:

- repeated across multiple entries
- tied to a decision
- useful for future work
- relevant to sales, delivery, platform, or governance
- a source of risk
- a validated assumption
- a reusable workflow
- a client-facing pattern
- a public narrative opportunity
- something the AI Cofounder should consistently remember

---

## Promotion destinations

| Destination | Use when the entry contains |
| --- | --- |
| Governance | Rules, standards, decisions, risks, review cadences |
| Narrative | Founder story, positioning, market lessons, content ideas |
| Pipeline | Sales signals, objections, pricing insights, discovery improvements |
| Delivery | Implementation lessons, workflows, templates, checklists |
| Platform | Product requirements, automation ideas, system patterns |
| Knowledge Graph | Concepts, relationships, recurring patterns, definitions |
| Task system | Specific work that must be done |
| ADR or decision record | Architecture or major operating decisions |

---

## Promotion workflow

Use this process.

1. **Identify**  
   Find useful material in a daily log, weekly review, or monthly theme.

2. **Classify**  
   Decide what type of item it is: decision, assumption, risk, task, template, SOP, requirement, or narrative asset.

3. **Redact**  
   Remove sensitive or unnecessary private information.

4. **Extract**  
   Rewrite the item in a clean, reusable format.

5. **Assign destination**  
   Choose the documentation system or operating system where it belongs.

6. **Link source**  
   Preserve the source log reference when possible.

7. **Create or update asset**  
   Add the promoted material to the correct page, task, template, or system.

8. **Review**  
   Confirm that the promoted item is accurate and useful.

---

## Promotion decision tree

Ask:

1. Is this a committed choice?  
   Promote to **Decision Extraction** or Governance.

2. Is this an unproven belief?  
   Promote to **Assumption Extraction**.

3. Is this something that could go wrong?  
   Promote to **Risk Extraction** or Governance.

4. Is this work that must be done?  
   Promote to the task system.

5. Is this a repeatable process?  
   Promote to Delivery or Governance as an SOP.

6. Is this useful for sales?  
   Promote to Pipeline.

7. Is this useful for public explanation?  
   Promote to Narrative.

8. Is this a product or automation requirement?  
   Promote to Platform.

9. Is this a recurring concept?  
   Promote to Knowledge Graph.

## Promotion record format

Use this format when promoting an item.

```md
## Promoted item

[Short name]

## Source

[Founder Log entry, weekly review, or monthly review]

## Type

Decision / Assumption / Risk / Task / Template / SOP / Requirement / Narrative Asset / Concept

## Destination

[Where it is being promoted]

## Extracted value

[What useful value was found?]

## Redaction applied

None / Client generalized / Person removed / Secret removed / Internal-only detail removed

## Follow-up

[What happens next?]

## Status

Proposed / Promoted / Reviewed / Published / Archived
````

## Example

Raw log note:

> We keep explaining discovery differently each time. We need a standard discovery script for SMB AI implementation calls.

Promoted item:

```md
## Promoted item

SMB AI Implementation Discovery Script

## Source

Founder Log daily entry

## Type

Template / Sales Asset

## Destination

Pipeline / Discovery System

## Extracted value

Repeated founder friction showed that discovery needs a standard question flow.

## Redaction applied

None

## Follow-up

Create a discovery script covering business context, workflow pain, current tools, manual tasks, automation opportunities, budget, urgency, and decision process.

## Status

Promoted
```

## Weekly promotion review

During weekly review:

* identify promotion candidates
* classify each candidate
* redact sensitive details
* promote quick wins immediately
* defer larger assets to tasks
* mark unresolved items for monthly review

---

## Monthly promotion review

During monthly review:

* look for repeated themes
* promote confirmed doctrine
* retire stale ideas
* consolidate duplicate assets
* update navigation if new docs are needed
* identify platform requirements from real work
* identify narrative assets from founder learning

---

## Quality checklist

Before promoting an item, confirm:

* Is the source clear?
* Is the value durable?
* Is the destination correct?
* Has sensitive information been removed?
* Is the promoted item written cleanly?
* Does it need human verification?
* Is it linked back to the original context?
* Will this improve future execution?

---

## Rule of thumb

Do not promote everything.

Promote what improves future decisions, future execution, future sales, future delivery, future governance, or future AI assistance.
