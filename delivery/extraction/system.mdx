# Asset Extraction System

The Asset Extraction System turns delivery work into reusable business infrastructure.

Every engagement should produce more than a client outcome. It should also produce learning that improves future sales, delivery, platform development, or governance.

---

## Purpose

The Asset Extraction System exists to capture:

- reusable templates
- repeatable workflows
- standard operating procedures
- delivery checklists
- sales insights
- product requirements
- risk controls
- documentation patterns
- implementation lessons

---

## Extraction points

Extract assets:

- after discovery
- after proposal
- after implementation
- after testing
- after handoff
- after client review
- during monthly review

---

## Asset types

| Asset type | Example |
| --- | --- |
| Template | Discovery summary template |
| SOP | Client access intake process |
| Checklist | Pre-handoff testing checklist |
| Script | Follow-up automation sales script |
| Product requirement | Need for reusable workflow mapping module |
| Governance rule | No client credentials in chat logs |

---

## Definition of done

Asset extraction is working when every meaningful engagement improves the next engagement.
