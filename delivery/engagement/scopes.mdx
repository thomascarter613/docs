# Engagement Scopes

Engagement Scopes define what is included and excluded in common client offers.

Clear scope protects both the client and Applied Innovation Corp.

---

## Scope principles

A good scope should be:

- specific
- outcome-oriented
- limited enough to deliver
- clear about exclusions
- connected to success criteria
- written before implementation begins

---

## AI Workflow Audit scope

Includes:

- workflow discovery
- current-state mapping
- friction analysis
- automation opportunity summary
- risk notes
- recommended next steps

Excludes:

- implementation
- custom software development
- ongoing support
- third-party subscription costs

---

## AI Implementation Sprint scope

Includes:

- one defined workflow
- implementation plan
- configuration or automation build
- testing
- documentation
- handoff

Excludes:

- unlimited revisions
- unrelated workflows
- custom platform development unless stated
- support beyond agreed period

---

## Operating System Buildout scope

Includes:

- workflow architecture
- documentation structure
- system mapping
- governance rules
- implementation roadmap

Excludes:

- full enterprise transformation
- unmanaged scope expansion
- legal or compliance certification

---

## Rule of thumb

If it is not written into scope, it should not be assumed included.
