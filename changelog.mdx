# Changelog

This page records meaningful changes to Monad-Workspace documentation, operating systems, governance, and architecture records.

The changelog exists so the documentation history remains visible.

---

## Changelog policy

Record changes when they affect:

- page structure
- navigation
- canonical systems
- governance rules
- ADRs
- operating rhythm
- templates
- public positioning
- platform architecture
- documentation standards

Do not record every typo fix unless it changes meaning.

---

## Entry format

Use this format:

```md
## YYYY-MM-DD

### Added

- 

### Changed

- 

### Deprecated

- 

### Removed

- 

### Fixed

- 
```

Only include sections that apply.

## Maintenance rules

1. Add changelog entries for meaningful documentation changes.
2. Use reverse chronological order.
3. Keep entries concise.
4. Link to related ADRs or pages when useful.
5. Do not use the changelog as a task list.
6. Do not include private or sensitive information.
7. Record structural changes clearly.

---

## Definition of done

The changelog is working when a reader can understand how the docs evolved without reading every commit.
