# Domains

This page defines the major domains in Monad-Workspace.

A domain is a coherent area of responsibility within the documentation and operating system.

Domains help keep the docs organized, reduce sidebar clutter, and clarify where different kinds of work belong.

---

## Domain map

| Domain | Purpose | Canonical entry point |
| --- | --- | --- |
| Founder operating system | Captures founder memory, daily logs, reviews, extraction, and AI Cofounder context | [Founder Log](./founder-log/overview) |
| Architecture and ADRs | Preserves architecture decisions, decision status, review process, and supersession | [ADR Index](./adr/index) |
| Narrative and authority | Turns internal learning into public narrative, positioning, publishing, and authority assets | [Narrative Engine](./narrative/overview) |
| Pipeline and revenue | Turns market learning into discovery, sales, pricing, and pipeline review | [Pipeline Engine](./pipeline/overview) |
| Delivery | Turns sold work into client outcomes, reusable assets, and implementation learning | [Delivery Engine](./delivery/overview) |
| Platform | Turns validated needs into Monad-Workspace modules, lineage, and platform templates | [Platform Engine](./platform/overview) |
| Governance | Defines standards, decision filters, operating rhythm, documentation control, and human verification | [Governance](./governance/overview) |
| Knowledge graph | Structures concepts, patterns, lineage, and reusable knowledge | [Knowledge Graph](./knowledge/overview) |

---

## Founder operating system

The Founder Operating System preserves the founder’s operating memory.

It includes:

- Founder Log
- Daily Logs
- Weekly Reviews
- Monthly Themes
- AI Cofounder Handoff
- Logging Standard
- Decision extraction
- Assumption extraction
- Risk extraction
- Redaction rules
- Promotion workflow

Use this domain when capturing, reviewing, or promoting founder context.

---

## Architecture and ADRs

The Architecture and ADR domain preserves decisions that shape Monad-Workspace.

It includes:

- ADR Index
- ADR Process
- ADR Status Model
- ADR Supersession Guide
- ADR Review Checklist
- Architecture Decisions
- ADR Template

Use this domain when a decision affects runtime, tooling, architecture, manifests, state, graph behavior, governance, or platform direction.

---

## Narrative and authority

The Narrative and Authority domain turns internal learning into external clarity.

It includes:

- Narrative Engine
- Founder Narrative
- Publishing System
- Publishing Cadence
- Publishing Templates
- Positioning System
- Positioning Framework
- Authority System
- Authority Assets

Use this domain when creating public-facing language, content, positioning, or trust-building material.

---

## Pipeline and revenue

The Pipeline and Revenue domain turns market contact into qualified opportunities.

It includes:

- Pipeline Engine
- Discovery System
- Discovery Questions
- Discovery Scripts
- Sales System
- Sales Script
- Sales Pricing
- Pricing Framework
- Pricing Models
- Pipeline Review System
- Pipeline Metrics

Use this domain when working on lead generation, discovery, offers, objections, pricing, and revenue learning.

---

## Delivery

The Delivery domain turns client commitments into fulfilled outcomes.

It includes:

- Delivery Engine
- Client Delivery System
- Client Delivery Workflows
- Asset Extraction System
- Asset Extraction Templates
- Engagement Templates
- Engagement Scopes

Use this domain when planning, delivering, reviewing, or extracting assets from client work.

---

## Platform

The Platform domain turns operating needs into product architecture.

It includes:

- Platform Engine
- Module Lineage
- Module Lineage Map
- Platform Templates
- Module Templates

Use this domain when defining Monad-Workspace modules, platform requirements, technical systems, and product architecture.

---

## Governance

The Governance domain protects decision quality, documentation quality, and human oversight.

It includes:

- Governance Overview
- Decision Filters
- Decision Filter Examples
- Documentation Standards
- Documentation Templates
- Weekly Operating Rhythm
- Daily Operating Rhythm

Use this domain when defining standards, review cadence, decision controls, or AI verification rules.

---

## Knowledge graph

The Knowledge Graph domain structures reusable meaning.

It includes:

- Knowledge Graph Overview
- Core Concepts
- Derived Concepts
- Patterns
- Pattern Library
- Lineage
- Lineage Map

Use this domain when turning repeated ideas, concepts, decisions, and patterns into structured knowledge.
