# Platform Templates Overview

Platform Templates provide reusable structures for defining Monad-Workspace modules, systems, requirements, and implementation ideas.

They help keep platform planning consistent and traceable.

---

## Purpose

Platform Templates exist to:

- standardize module definitions
- connect requirements to evidence
- reduce vague product planning
- support ADR creation
- improve implementation readiness
- preserve reasoning

---

## Template types

| Template | Purpose |
| --- | --- |
| Module Template | Define a platform module |
| Requirement Template | Define a feature or capability |
| Integration Template | Define native tool integration |
| Policy Template | Define governance behavior |
| AI Context Template | Define context behavior for AI-assisted work |

---

## Template rule

Every platform template should include:

- purpose
- source
- problem
- users
- requirements
- dependencies
- risks
- related ADRs
- status

---

## Definition of done

Platform Templates are working when platform ideas become structured enough to evaluate, implement, or reject.
