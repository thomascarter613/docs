# Architecture Decisions

Architecture Decisions are the technical and structural choices that shape Monad-Workspace.

The canonical list of Architecture Decision Records lives in the ADR Index. This page explains how those decisions relate to the Platform Engine.

---

## Relationship to ADRs

The ADR system records durable decisions.

The Platform Architecture section explains how those decisions affect implementation.

Use:

- `adr/index` for the canonical decision list
- individual ADR pages for decision rationale
- this page for platform-level interpretation
- `platform/lineage/map` for module relationships

---

## Current architectural baseline

The accepted Monad-Workspace architecture baseline includes:

- Rust-first runtime
- native tool coordination
- multi-crate Rust workspace
- latest stable Rust during early development
- early Tokio adoption
- canonical `monad.toml` manifest
- `monad.lock` and `.monad/` operational state
- multiple graph output formats

| Category | Purpose |
