# Module Lineage Overview

Module Lineage records where platform modules come from, why they exist, and what evidence supports them.

It prevents Monad-Workspace from becoming a collection of disconnected ideas.

---

## Purpose

Module Lineage exists to connect each module to:

- source problem
- founder need
- client need
- platform requirement
- ADR
- implementation status
- related systems
- future roadmap

---

## Lineage questions

For every module, ask:

- Why does this module exist?
- What problem does it solve?
- Where did the need appear?
- What decision supports it?
- What depends on it?
- What would happen if it did not exist?

---

## Module record format

| Field | Description |
| --- | --- |
| Module | Name of the module |
| Source | Where the idea came from |
| Problem | What problem it solves |
| Evidence | Why it matters |
| Related ADR | Decision record |
| Status | Proposed, active, deferred, deprecated |
| Dependencies | Related modules or systems |

---

## Definition of done

Module Lineage is working when every major module has a traceable reason to exist.
