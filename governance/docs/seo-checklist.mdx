# SEO checklist

Use this checklist to improve discoverability for Monad-Workspace documentation.

SEO is not only for external search engines. Good titles, descriptions, headings, and links also improve internal search and AI retrieval.

---

## Page metadata

Confirm each important page has:

- clear `title`
- specific `description`
- relevant `tags`
- current `updated` date
- accurate `status`
- useful `parents` and `children`
- unique title
- unique description

---

## Title standard

A good title should be:

- specific
- short
- recognizable
- written in sentence case
- aligned with the page purpose

Avoid generic titles like:

- Overview
- System
- Template
- Notes

Prefer:

- Founder Log
- Pipeline Engine
- ADR Review Checklist
- Pricing Framework
- Documentation Standards

---

## Description standard

A good description should explain:

- what the page is
- who it helps
- what problem it solves
- how it fits the docs

Keep descriptions specific.

---

## Heading structure

Use a clear heading hierarchy:

```md
# Page title

## Main section

### Subsection
````

Do not skip from `#` to `###`.

---

## Internal links

Add links to:

* parent pages
* child pages
* related governance pages
* relevant ADRs
* templates
* checklists
* canonical entry points

Use descriptive link text.

---

## Search terms

Include the terms users are likely to search for.

Examples:

* Founder Log
* AI Cofounder
* ADR
* governance
* pipeline
* pricing
* delivery
* platform
* knowledge graph
* decision filters
* redaction

---

## Search boosts

Use `boost` sparingly.

Boost only canonical entry points and high-value reference pages.

Recommended boosted pages:

* `quickstart`
* `search`
* `domains`
* `adr/index`
* `founder-log/overview`
* `governance/overview`
* `pipeline/overview`
* `delivery/overview`
* `platform/overview`
* `knowledge/overview`

---

## Definition of done

SEO is working when readers, search engines, and AI systems can understand what each page is about and find the right page quickly.
