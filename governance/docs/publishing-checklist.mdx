# Publishing checklist

Use this checklist before publishing new or materially changed documentation.

Publishing means the page is ready to guide real work or be read by others.

---

## Content checklist

Confirm the page has:

- clear title
- useful description
- accurate status
- current `updated` date
- correct owner
- correct review cycle
- clear purpose
- practical operating guidance
- related links where useful
- definition of done

---

## Structure checklist

Confirm the page:

- uses sentence case headings
- uses short sections
- uses tables for comparisons
- uses numbered lists for ordered processes
- uses bullets for unordered guidance
- avoids unnecessary repetition
- fits the correct domain
- does not duplicate another page

---

## Safety checklist

Confirm the page does not include:

- secrets
- API keys
- credentials
- private client information
- sensitive personal information
- unsupported claims
- unapproved pricing commitments
- unverified legal or financial conclusions
- internal speculation presented as fact

---

## AI-generated content checklist

If AI helped create the page, confirm:

- facts were checked
- the page reflects founder intent
- draft notes were not treated as final doctrine
- claims are not invented
- uncertainty is marked
- the page does not overstate maturity
- human verification is complete

---

## Navigation checklist

Before adding a page to `docs.json`, confirm:

- the file path exists
- the page belongs in the selected section
- the title is distinct enough for search
- the sidebar will not become unnecessarily cluttered
- nested grouping would not be clearer
- the page should be discoverable

---

## Definition of done

A page is ready to publish when it is useful, safe, accurate, aligned, and easy to find.
