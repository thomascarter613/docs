# Documentation Standards

Documentation Standards define how Monad-Workspace docs should be written, organized, reviewed, and maintained.

Docs are not decoration. They are operating infrastructure.

---

## Purpose

Documentation Standards exist to ensure docs are:

- useful
- current
- structured
- searchable
- linked
- reviewable
- safe to reuse
- helpful to humans and AI systems

---

## Page standard

Every stable page should include:

- frontmatter
- clear title
- description
- purpose
- core concepts
- operating rules
- definition of done
- review cadence
- owner
- links to related pages where useful

---

## Writing standard

Docs should be:

- direct
- specific
- practical
- structured
- free of unnecessary hype
- clear about status
- clear about uncertainty
- grounded in real decisions or workflows

---

## Maintenance standard

Every page should have:

- owner
- review cycle
- status
- updated date
- parent and child references where useful

---

## Status values

Recommended statuses:

- draft
- proposed
- active
- stable
- deprecated
- superseded

---

## Definition of done

Documentation standards are working when docs help the founder, collaborators, and AI systems continue work without reconstructing context from scratch.
