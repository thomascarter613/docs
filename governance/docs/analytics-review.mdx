# Analytics review

Analytics Review turns documentation usage into a maintenance backlog.

Use analytics to learn what readers search for, which pages matter, and where the docs are unclear.

---

## Purpose

Analytics Review exists to:

- identify popular pages
- identify missing pages
- find confusing search terms
- improve navigation
- improve page titles and descriptions
- prioritize documentation work
- reduce stale or unused content

---

## Monthly review inputs

Review:

- top pages
- low-traffic but important pages
- search queries
- failed or low-click searches
- assistant questions
- feedback submissions
- pages with high exits
- pages that should be linked but are not

---

## Review questions

Ask:

- What are readers looking for?
- Which pages are most important?
- Which pages are hard to find?
- Which search terms reveal missing docs?
- Which pages need better titles?
- Which pages should be linked from overview pages?
- Which docs should be promoted in navigation?

---

## Output

A completed analytics review should produce:

- page updates
- navigation improvements
- new docs tasks
- search tuning changes
- changelog entries
- stale page review candidates

---

## Definition of done

Analytics Review is working when real usage data improves the structure, clarity, and usefulness of the docs.
